# [TARGET-002-DB] · Refactor annual_targets schema for dynamic targets

**Timestamp**: 2026-03-16 22:30 Local Time  
**Author**: Ralph  

## Why
The previous schema only supported two hardcoded targets per KPI. A normalized structure is required to support an arbitrary number of targets.

## What Changed
- `src/data_access/setup.py`: Updated to create `kpi_annual_target_values` table and migration logic. Updated periodic target tables to allow `target_number > 2`.
- `scripts/migrate_dynamic_targets.py`: (New) Utility script to perform the initial migration of existing target data.
- `databases/db_kpi_targets.db`: New table `kpi_annual_target_values` created and populated.
- Periodic Databases (`db_kpi_days.db`, etc.): Constraints updated to support dynamic target numbers.

## The "Why"
The application required a more professional and flexible architecture to handle complex organizational structures. Moving to a recursive hierarchy allows for unlimited nesting of KPIs, while the UI refactor provides a consistent "Master-Detail" experience across all management tabs. Decoupling CSV export from data entry ensures system stability and user-controlled data flow.

## Verification
- **Architecture**: Migrated to `kpi_nodes` table with recursive `parent_id`.
- **UI Consistency**: Refactored Plants, Data Management, and Results tabs to use unified explorer/detail layouts.
- **Entry Stability**: Implemented `FocusOut` validation in Target Entry to prevent UI jitter during formula recalculation.
- **Data Flow**: Verified that CSV exports are now explicitly user-triggered in the Data Management Center.
- **Migration**: Confirmed that legacy Group/Subgroup data was correctly migrated to the new recursive node structure.

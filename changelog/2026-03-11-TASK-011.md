## The "Why"
To ensure data integrity and user control, the system was updated to prioritize manual entries over calculated values while still allowing for complex formula logic. The integration of the Visual Node Editor directly into the data entry workflow provides a seamless experience for defining and overriding KPI targets. Additionally, moving the save operation to a background thread ensures the UI remains responsive even during heavy batch calculations.

## Verification
- **Manual Override**: Refined `_on_target_value_change` to handle user-initiated changes vs automated updates.
- **Node Editor Integration**: Added a "Nodes..." button to each KPI entry frame that launches the visual editor and populates the formula field.
- **UI Responsiveness**: Wrapped the `save_annual_targets` call in a `threading.Thread` with appropriate `self.after` callbacks for UI notifications.
- **Persistence**: Verified that `is_manual` and formula flags are correctly captured and sent to the backend for storage.

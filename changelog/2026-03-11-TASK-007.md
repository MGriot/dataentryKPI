## Timestamp
[12:30:00]

## The "Why"
To provide a user-friendly way to define complex KPI calculations, a visual node editor was implemented. This allows users to drag and drop KPI inputs, constants, and operators, and connect them to form a calculation DAG. This is far more intuitive than manual formula strings and reduces syntax errors.

## Verification
- **Implementation**: Created `src/gui/node_editor.py` containing a `NodeEditorDialog` based on `tk.Canvas`.
- **Interactions**: Implemented node dragging, edge creation via socket dragging, and node deletion.
- **Node Types**: Supported `KpiInput` (with selection dialog), `Constant`, `Operator` (with selection dialog), and `Output`.
- **Serialization**: Integrated with the `KpiDAG` logic to load and save formula state as JSON.

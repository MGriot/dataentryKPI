# [UI-PARITY] · Streamlit-Tkinter Parity Audit Report

**Timestamp**: 2026-03-16 23:55 Local Time  
**Author**: Ralph  

## Executive Summary
The Streamlit interface has the core functionality but lacks the visual polish and several advanced management features present in the Tkinter version. The migration needs to focus on navigation consistency and missing configuration modules.

## Gap Analysis

### 1. KPI Management (High Priority)
- **Missing**: "Global Splits" management page. Tkinter allows defining shared repartition templates; Streamlit has no UI for this.
- **Incomplete**: "Master/Sub Links" UI. Streamlit uses a simple selectbox; Tkinter shows a list/tree of masters and their linked subs with weights.
- **UX**: Icon-coded tree navigation (🏢 for Groups, 📂 for Subgroups, 📊 for Indicators) is missing in Streamlit.

### 2. Results Analysis (Medium Priority)
- **Multi-Year**: Streamlit only allows selecting ONE year or "All". Tkinter's logic for multi-year comparison is more robust.
- **Day Scale**: The day-scale visualization on charts needs better formatting (currently standard Plotly, needs manufacturing-specific date handling).
- **Selector**: The "Single KPI Focus" vs "Global Comparison" toggle works but the "Global Comparison" needs a proper dashboard-style summary instead of 10 sequential charts.

### 3. KPI Detail Preview (New Requirement)
- **Missing**: Real-time formula/node preview in the right panel when editing a KPI.

## Task Decomposition for UI-MIGRATE
The `UI-MIGRATE` task will be split into:
1. `UI-MIGRATE-SPLITS`: Implement Global Splits page.
2. `UI-MIGRATE-LINKS`: Implement hierarchical Master/Sub management.
3. `UI-MIGRATE-ANALYSIS`: Multi-year selection and improved chart scaling.
4. `UI-MIGRATE-PREVIEW`: Real-time KPI formula preview.

# Changelog: 2026-03-15-TASK-027

## Task ID & Title
TASK-027: Multi-Plant Target Entry & Backend Save Refactor

## Timestamp
[15:30:00]

## The "Why"
Efficiency was hindered by the need to save targets individually for each plant. Furthermore, the distribution logic contained a critical bug where Target 2 would mirror Target 1 during formula evaluation, and repartitioning was not correctly triggered for derived sub-KPIs.

## Verification
- **UI**: Added "Apply to All Plants" checkbox in `TargetEntryTab`.
- **Backend**: Refactored `save_annual_targets` into a bulk-capable operation with fixed `target_num` resolution in the Node Engine.
- **Distribution**: Corrected Phase 3 logic to properly derive sub-KPI values from masters across multiple plants.
- **Repartitions**: Ensured `calculate_and_save_all_repartitions` is triggered for all affected KPIs at the end of the save process.

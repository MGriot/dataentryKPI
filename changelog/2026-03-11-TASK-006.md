TASK-006: Node Engine: Define Serialization Schema

## Timestamp
[12:00:00]

## The "Why"
To transition from static `eval`-based formulas to a dynamic and visual calculation engine, a robust serialization format is required. The JSON-based DAG (Directed Acyclic Graph) schema allows for complex, multi-level KPI dependencies while being easy to store in the database and visualize in a node editor.

## Verification
- **Implementation**: Created `src/core/node_engine.py` with `KpiNode`, `KpiEdge`, and `KpiDAG` classes.
- **Serialization**: Implemented `to_json` and `from_json` methods.
- **Evaluation**: Implemented a recursive evaluation engine with support for basic operators (+, -, *, /, pow, min, max, avg) and external KPI resolution.
- **Testing**: Created and ran `test_node_engine.py` which successfully verified DAG creation, serialization, deserialization, and multi-node evaluation.

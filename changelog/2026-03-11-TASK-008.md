## The "Why"
To make the visual formulas operational, the core `save_annual_targets` logic was updated to recognize and evaluate the JSON-based DAGs. By integrating this into the existing iteration loop, the system now supports multi-level dependencies between KPIs, where one calculated KPI can serve as an input to another, all while maintaining backward compatibility with legacy string-based formulas.

## The "Why"
To operationalize the standardized splits, the core repartition logic needed to be updated to respect the `global_split_id` if present. This allows users to define a single distribution template (e.g., 100% in January) and apply it globally, simplifying maintenance and ensuring consistency across plants.

- **Testing**: Created and ran `test_global_repartition.py` which confirmed that:

    2. Global values override local `repartition_logic`, `repartition_values`, `distribution_profile`, and `profile_params`.
    3. The periodic target calculations (daily, monthly, etc.) correctly reflect the global template.

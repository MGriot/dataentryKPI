## Task ID & Title
TASK-017: Hybrid Formula System & Unified Calculation Engine

## Timestamp
[11:15:00]

## The "Why"
Users needed more flexibility in defining KPI calculations. While the visual node editor is powerful for complex logic, simple arithmetic is often faster to type as a string. This update provides a "best of both worlds" approach, allowing users to choose their preferred method while ensuring both are evaluated reliably by a unified engine.

## Verification
- **KPI Editor**: Updated `IndicatorSpecEditorDialog` with a mode toggle and a `KPI Picker` for inserting `[ID]` references.
- **UI Recalculation**: Enhanced `TargetEntryTab` to evaluate both Node DAGs and String Expressions in real-time.
- **Backend Integrity**: Updated `save_annual_targets` in `annual.py` to support string formula evaluation during the persistent calculation phase.
- **Security**: Implemented safe `eval` with restricted built-ins and regex-based ID replacement.

# Changelog: 2026-03-12-FIX-002

## Task ID & Title
FIX-002: Global Dashboard: Fix Multi-Year Timeline Aggregation

## Timestamp
[11:45:00]

## The "Why"
When selecting "All" years in the Global Dashboard, the chart was incorrectly aggregating data by period name (e.g., all "Januaries" combined) instead of showing a continuous historical timeline. This prevented users from seeing multi-year trends correctly.

## Verification
- **Timeline Logic**: Implemented `timeline_label` generation in `load_dashboard_data`.
- **Sorting**: Added `_sort_dashboard_df` to handle chronological ordering of months, quarters, and weeks across years.
- **UI Testing**: Verified that selecting "All" years now displays a linear timeline (e.g., 2023-Jan, 2023-Feb... 2024-Jan) without overlapping data points.
